Gerenciando seu blog com Jekyll, GitHub e microdata

20-07-2014 jekyll, microdata, github pages nandotorres

Se você quer começar a utilizar Jekyll, ou mesmo se você já usa, mas quer automatizar tarefas como criação de textos e deploy no GitHub pages, este post pode ser útil para você.

É assumido que você já tenha o Jekyll instalado em seu computador, assim com uma conta gratuita no GitHub. Caso você precise instalar e configurar o git em seu computador, siga esses passos antes de continuar.

Versionando seu blog jekyll com GitHub

Suponha que tenha criado seu novo blog com jekyll: 

$ jekyll new meublog
$ cd meublog && ls -la
drwxr-xr-x  12 fernando  staff   408 Jul 20 20:10 .
drwxrwxr-x  32 root      staff  1088 Jul 20 19:59 ..
-rw-r--r--   1 fernando  staff     5 Jul 20 19:59 .gitignore
-rw-r--r--   1 fernando  staff   435 Jul 20 19:59 _config.yml
drwxr-xr-x   5 fernando  staff   170 Jul 20 19:59 _includes
drwxr-xr-x   5 fernando  staff   170 Jul 20 19:59 _layouts
drwxr-xr-x   3 fernando  staff   102 Jul 20 19:59 _posts
-rw-r--r--   1 fernando  staff   470 Jul 20 19:59 about.md
drwxr-xr-x   3 fernando  staff   102 Jul 20 19:59 css
-rw-r--r--   1 fernando  staff  1292 Jul 20 19:59 feed.xml
-rw-r--r--   1 fernando  staff   451 Jul 20 19:59 index.html

Agora você está pronto para começar a blogar! O primeiro passo é alterar o arquivo _config.yml e definir as informações do seu blog, como titulo, descrição e endereços de suas redes sociais no twitter e github. Nesse ponto já se torna interessante começar a versionar o código fonte do seu blog. Mais ainda, é hora de enviá-lo para um repositório para manter seu código a salvo.

No GitHub, crie um novo repositório para o código fonte do seu blog. Você pode querer manter seu código fonte em um repositório privado caso não queira compartilhar seus temas ou plugins customizados com a comunidade do GitHub.

Nesse repositório manteremos os arquivos de configuração, templates e arquivos de posts. O Jekyll irá compilar seu site em um diretório _site mas ele não será versionado pois por padrão o Jekyll cria o arquivo .gitignore contendo uma instrução para manter este diretório fora do controle de versão. Esse diretório será mantido em outro repositório.

Supondo que criamos o repositório meublog no GitHub, teremos então o endereço git deste repositório em https://github.com/suacontanogithub/meublog.git.

De volta ao seu computador, no diretório raiz do seu blog (meublog/) faça o commit do código atual e faça também o primeiro push para o GitHub.

git init
git add .
git commit -m "Configurando o config.yml"
git remote add origin https://github.com/suacontanogithub/meublog.git
git push -u origin master

Pronto! Agora o código fonte do seu blog ou site está versionando. A cada conjunto de alterações que fizer, você deve enviar as alterações para o GitHub.

git commit -m "Alterando o sidebar do meu novo tema"
git push -u origin master

Gerando seu blog e fazendo deploy para GitHub Pages

O GitHub oferece um serviço gratuito de hospedagem de conteúdo estático (HTML, CSS, Javascript, imagens) para seus usuários chamado GitHub Pages. Basta ter uma conta no GitHub e você está pronto para utilizar.

Acesse sua conta no GitHub e crie um novo repositório, obrigatoriamente com o nome suaconta.github.io - por exemplo, no meu caso criei um repositório chamado nandotorres.github.io.

Volte ao diretório raiz do seu blog e gere seu site com o comando jekyll build: 

$ jekyll build
Configuration file: /app/meublog/_config.yml
            Source: /app/meublog
       Destination: /app/meublog/_site
      Generating... 
                    done.

Após o primeiro build, seu site é gerado no subdiretório _site. Vamos versioná-lo, mas ao contrário do diretório do código fonte, que versionamos no repositório meublog, vamos versionar seu site no repositório suaconta.github.io.

Mova para o diretório _site e inicie o repositório:

$ cd _site   # não se esqueça de se mover para dentro do diretório _site
$ git init
$ git add .
$ git commit -m "Bem vindos ao meu blog"
$ git remote add origin https://github.com/suacontanogithub/suacontanogithub.github.io.git
$ git push -u origin master

Acesse seu site e veja se tudo está no ar: http://suacontanogithub.github.io. Agora é hora de deixar tudo mais organizado.

Script para gerenciar seu site Jekyll com GitHub e GitPages

A cada vez que você escrever um post ou fizer alterações no seu blog você deve repetir os seguintes passos:

  1. Adicionar os novos arquivos e commitar o codigo fonte do seu blog
  2. Fazer push do seu código para o repositório
  3. Gerar seu site jekyll build
  4. Entrar no diretório _site, adicionar os novos arquivos gerados e commitar seu site
  5. Enviar a nova estrutura do seu site para o repositório com nome da sua github page

Para automatizar essa tarefa você pode escrever um script shell ou batch ou usar algum pronto. Se optar pela segunda opção, disponibilizo um script bash (para Linux e Mac OSX por enquanto) que realiza essa tarefa.

Faça o download do script de administração do Jekyll com GitHub Pages e siga as instruções de instalação e uso.

Script para criar novos posts no Jekyll

Para automatizar a tarefa de criar novos posts eu utilizo um script em Python disponibilizado por Josh Branchaud em seu blog.

Basta baixar o arquivo e salvá-lo em seu diretório home (~/jekyll-post), alterar a permissão do arquivo chmod 755 ~/jekyll-post, inserir um alias para ele em seu ~/.bash_profile: alias jekyll-post="~/jekyll-post" salvar e atualizar sua sessão: source ~/.bash_profile.

Pronto! Agora é só ir para o diretório raiz do seu site e executar o comando:

$ jekyll_post "Como não desenvolver seu app em HTML5" -D _posts
New Jekyll post "Como não desenvolver seu app em HTML5" has been created -- _posts/2014-07-21-Como-no-desenvolver-seu-app-em-HTML5.md

Abra o arquivo _posts/2014-07-21-Como-no-desenvolver-seu-app-em-HTML5.md e você está pronto para inserir conteúdo no template gerado:

---
layout: post
title: Como não desenvolver seu app em HTML5
summary:
status: draft
hn-discussion:
category:
---

Finalizando

Com essas configurações em ordem a tarefa de blogar com Jekyll fica muito mais fácil. Essas foram as primeiras necessidades que tive assim que comecei a utilizar essa preciosa ferramenta. A medida que forem surgindo mais avanços irei compartilhar as dicas aqui. Até lá!
